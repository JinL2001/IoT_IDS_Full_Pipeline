02 — Exploratory Data Analysis

What the Data Looks Like

Class Imbalance

Both datasets are heavily imbalanced toward attack traffic. Dataset 1 (cleaned) is roughly 79% attack vs 21% normal. Dataset 2 (full metadata) is even more skewed at 92% attack vs 8% normal. This is realistic — in a real network, botnet floods generate far more packets than benign use — but it means models need class weighting to avoid ignoring the minority "normal" class entirely.

Correlation Structure — 4 Perfectly Redundant Features

The correlation heatmap revealed 4 feature pairs with r = 1.0000, meaning one column is an exact copy of another. These were safely dropped, reducing the feature space from 23 to 19 with zero information loss.

Redundant Pairs (r = 1.000)
H_L0.1_weight ↔ MI_dir_L0.1_weight  |  H_L0.1_mean ↔ MI_dir_L0.1_mean  |  H_L0.1_variance ↔ MI_dir_L0.1_variance  |  HH_jit_L0.1_weight ↔ HH_L0.1_weight

Skewness — 12 Features Need Log Transform

12 out of 23 features have |skewness| > 1, meaning their distributions have very long tails (some exceeding skew = 13). Without transformation, gradient-based models like Logistic Regression and MLP would struggle. A log1p(|x|) transform was applied to all of these before standardization.

Attack Distribution (Dataset 2)

The 399K-row sample from Dataset 2 contains three attack families: mirai (largest), gafgyt, and Normal. The 9 subtypes are: udp, tcp, scan, syn, ack, combo, junk, udpplain, and Normal. UDP flood is the dominant subtype, consistent with Mirai's real-world behavior of using UDP floods for DDoS.

04 — Multiclass Classification

Attack Family & Subtype Identification

3-Class: Normal vs Mirai vs Gafgyt

The HGB model achieves perfect classification across all three families — F1 = 1.00 for Normal, gafgyt, and mirai on the 60K test set. This is a remarkable result: the two major botnet families leave completely non-overlapping signatures in the 19 flow statistics. A network IDS can not only detect an attack is happening, but immediately identify which botnet family is responsible.

9-Class: Attack Subtypes (UDP, TCP, SYN, ACK, Combo, Junk, Scan, UDPPlain + Normal)

The 9-subtype classification is nearly perfect as well. Almost every subtype achieves F1 = 1.00, with one minor exception:

Only Notable Imperfection
TCP subtype achieves precision = 0.99 (slightly below 1.00) while recall stays at 1.00. This means a tiny number of non-TCP flows are occasionally classified as TCP, but no actual TCP flows are missed. This is the only non-perfect metric in the entire multiclass evaluation.

Overall 9-class accuracy on the test set is 100% — the model perfectly separates all 9 attack subtypes plus normal traffic.

MI_dir_L0.1_weight — the packet count in the 10-second window — accounts for 32% of the RF's decision power. This makes intuitive sense: botnets generate enormous packet floods (especially UDP/SYN floods), so the raw volume statistic is the single most discriminative feature.

MLP Permutation Importance — Model-Agnostic Verification

The permutation importance analysis on the MLP (shuffling each feature and measuring F1-Macro drop) confirms the RF ranking with some additions: MI_dir_L0.1_weight, HH_jit_L0.1_mean, MI_dir_L0.1_variance, HH_L0.1_mean, and HH_L0.1_magnitude are the top 5.

Surprising Result — Top 10 Beats Full Feature Set
Training MLP on only the top 10 permutation-important features achieved F1-Macro = 0.9995, vs 0.9993 with all 19 features. The extra 9 features add noise, not signal. A real-time IDS could safely use just 10 features, cutting computation by ~47%.

⚠️ PARTIALLY MET
Real-World Deployment Readiness
Latency goals are met (LogReg 0.6μs, MLP 5.7μs). However, the dataset has no temporal split — train/test come from the same capture. Real-world drift, new attack variants, and unseen devices remain untested.
⚠️ Important Caveat on Scores
The near-perfect scores (≥0.999) reflect that this is an in-distribution benchmark: training and test data come from the same 10-second capture windows and the same 9 devices. In production, a real IDS would face concept drift (evolving attack patterns), entirely new device types, and novel botnet families. The next step for this project should be a temporal train/test split to get a more honest real-world estimate.
08 — Key Observations

What We Learned

📦
Packet Volume is King
MI_dir_L0.1_weight (packet count) accounts for 32% of RF importance alone. Botnets are volumetric by nature — the sheer number of packets in a 10-second window is the strongest signal. This is consistent with how Mirai generates UDP floods.
✂️
4 Features Are Pure Duplicates
The H_* features are exact mathematical copies of the MI_dir_* features (r=1.000), and HH_jit weight equals HH weight exactly. Keeping them wastes computation and can mislead importance rankings. Removing them is strictly beneficial.
⚡
LogReg is Shockingly Competitive
Logistic Regression achieves F1-Macro = 0.9990 with a 0.577μs latency — 79× faster than RF and 10× faster than MLP. For edge devices with strict latency budgets, this simple model is a legitimate choice with minimal accuracy trade-off.
🌐
One Model Fits All Devices
A single trained model generalizes perfectly to all 9 IoT device types, from baby monitors to thermostats. This means a centralized IDS doesn't need per-device retraining — it can monitor a heterogeneous network with a single model.
🏷️
Mirai vs Gafgyt are Perfectly Separable
Despite both being IoT botnets with overlapping targets, the two botnet families produce completely non-overlapping statistical fingerprints. The HGB model separates them without a single misclassification, which is useful for incident response attribution.
🔟
10 Features Beat 19
The top-10 permutation-important features actually outperform the full 19-feature MLP (0.9995 vs 0.9993 F1-Macro). The extra 9 features add noise. A lean 10-feature model is both faster and more accurate — a key result for deployment.
09 — Conclusion

Summary

This project successfully demonstrates that IoT botnet attacks can be detected and classified with near-perfect accuracy using only 19 engineered network flow statistics extracted from 10-second windows — no raw packet inspection required. The pipeline is fast enough for real-time deployment even on constrained hardware.

All five research questions were answered affirmatively. Binary detection, botnet family identification, 9-way subtype classification, feature reduction, and cross-device generalization all achieved F1 scores of 0.999 or higher. The MLP is the recommended model for the final system: it offers the best balance of accuracy (F1-Macro 0.9993, AUC 1.000) and speed (5.66μs/sample).

The primary remaining limitation is the lack of a temporal holdout test: all evaluation is in-distribution. Future work should test on traffic captures from different time periods or new devices to validate real-world robustness against concept drift and novel attack variants.
